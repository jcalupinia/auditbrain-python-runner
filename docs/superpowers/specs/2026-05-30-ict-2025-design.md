## 0. Resumen ejecutivo

Construir la **primera herramienta de negocio del Portal Cliente de Audit Consulting Group**: una aplicación web que permite a un cliente externo armar incrementalmente su Anexo del Informe de Cumplimiento Tributario (ICT) 2025, subiendo documentos cuando los tiene disponibles, y descargar al final un Excel **100% idéntico al template oficial del SRI** con las celdas de input llenas y las fórmulas originales del template preservadas.

**UX clave:** una sola pantalla "dashboard" por proyecto ICT (similar al índice del Anexo), con botones de upload agrupados por anexo. Sesión persistente 90 días. Datos extraídos sobreviven a la eliminación de los archivos crudos (24h post-proceso por seguridad).

**Alcance MVP:** framework completo + **los 10 anexos** funcionando end-to-end (ÍNDICE, A1, A2, A3, A4, A5, A6, A7, A8, A9). Sin diferimiento — la herramienta sale completa el primer release.

**Insight clave**: tras analizar un ICT real ya lleno (ARCOLANDS CÍA. LTDA. 2024), confirmamos que:
- A1 requiere inserción dinámica de filas (subcuentas del balance bajo cada casillero F-101)
- A2 son 3 cuadros tabulares con dimensiones FIJAS
- A3 son 9 bloques con misma estructura, fórmulas Excel ya en template
- A4-A8 son combinaciones de las técnicas anteriores
- A9 es la más simple (9 casilleros fijos + datos kardex)

Tiempo estimado: ~4-5 semanas para entregar los 10 anexos en producción.

## 1. Decisiones tomadas

| # | Decisión | Valor | Razón |
|---|----------|-------|-------|
| 1 | UX del cliente | Dashboard único por proyecto ICT con uploads agrupados por anexo | Flujo realista: el cliente NO tiene todos los docs el mismo día |
| 2 | Persistencia de sesión | 90 días sin actividad | Ciclo anual de cumplimiento, espacio para iteración |
| 3 | Política de retención de archivos crudos | 24h post-procesado (heredado del portal) | Privacidad de datos sensibles |
| 4 | Política de retención de datos extraídos | 90 días en DB | Permite regenerar Excel sin re-subir |
| 5 | Política de retención del Excel generado | No se persiste, se regenera bajo demanda | Siempre refleja el estado actual de la sesión |
| 6 | Template del output | El Excel oficial SRI 2025 (semilla en repo) | Garantiza formato y fórmulas idénticas |
| 7 | Estrategia de llenado | Sobrescribir solo celdas de INPUT, preservar fórmulas y formato | Excel calcula las fórmulas al abrir, NO Python |
| 8 | F-101 format | PDF | Lo que el SRI emite a 100% de los clientes |
| 9 | Tolerancia a fallas de parseo | Estado por anexo: `empty / partial / ready` | Permite avance progresivo aún con datos incompletos |
| 10 | Alcance MVP | Framework + los 10 anexos completos (ÍNDICE, A1, A2, A3, A4, A5, A6, A7, A8, A9) | Entregar herramienta lista para producción sin diferimientos |
| 10b | Estrategia para A1 | Inserción dinámica de filas (cada casillero F-101 = 1 fila base + N subcuentas del balance) | Validado en ARCOLANDS: casillero 311 "Efectivo" tiene 6 subcuentas; A1 lleno crece de 40 a 197 filas |
| 10c | Estrategia para A3 | 9 bloques con misma estructura, solo llenar casilleros F-101 base | Las fórmulas de cálculo (porcentajes, totales) ya están en el template — Excel calcula solo |
| 10d | Estrategia para A8 | Único anexo que requiere parser XML (ATS/REOC del SRI) | Las 3 tablas A/B/C (pagos con CDI, sin CDI, reembolsos) tienen dimensiones variables por número de transacciones |
| 10e | Estrategia para A7 | Matrices tabulares por año (IR 3 años, ISD 5 años) con totales por fórmula | Cliente sube F-101 de años anteriores + F-108 + resoluciones SRI |
| 11 | Integración con portal cliente | NO entra al `tool_registry` genérico; es módulo dedicado `ict/` con endpoints propios | El ICT requiere sesión persistente y estado, no encaja en el modelo "subir-procesar-descargar" del registry |
| 12 | Reutilización código existente | `find_casillero_value`, `find_periodo` de `aud/obligaciones_fiscales/cedulas/base.py` + patrón openpyxl del `excel_assembler.py` | Ya probado en producción |
| 13 | Catálogo casillero F-101 ↔ cuenta contable | Diccionario Python hardcoded (extensible a DB futuro) | El catálogo SRI es estable; iteramos rápido |
| 14 | Categoría en el catálogo cliente | `CUMPLIMIENTO_TRIBUTARIO` | Reemplaza el `STUB_ECHO` o convive con él |
| 15 | Auth | Mismas 3 capas del portal cliente: JWT+rol, device, sid | Heredado, sin cambios |

### 2.3 Comportamiento UI

| Acción | Reacción del sistema |
|--------|----------------------|
| Cliente sube F-101 | Procesa inmediatamente, extrae casilleros, marca ✅ en A1 y A9 (reutiliza el mismo F-101) |
| Cliente sube Balance | Procesa, extrae cuentas+saldos, A1 pasa a 🟡 si faltaba algo o ✅ si está completo |
| Cliente sube Kardex | Procesa, A9 pasa a ✅ |
| Cliente click "Descargar" | Genera Excel ICT al vuelo con todos los datos actuales y descarga |
| Cliente sale y vuelve mañana | Login → ve su dashboard exactamente como lo dejó |
| Cliente edita datos contribuyente | Modal de edición, ÍNDICE se regenera |
| Pasan 90 días sin actividad | Sesión marcada `expired`, datos preservados 30 días extra (recuperable por staff) |

### 2.4 Estados visuales de un anexo

| Estado | Indicador | Significado |
|--------|-----------|-------------|
| `empty` | ⚪ Pendiente | No se ha subido ningún input para este anexo |
| `partial` | 🟡 Parcial | Al menos un input subido, faltan otros para completar |
| `ready` | ✅ Completo | Todos los inputs requeridos están listos |
| `error` | ❌ Error | El parseo del último input falló (con mensaje claro) |
| `locked` | 🔒 Próximamente | Anexo no implementado todavía en este MVP |

## 4. Modelos de datos

### 4.1 ICTSession (NEW)

```python
class ICTSession(Base):
    """Sesión persistente del proyecto ICT de un cliente.
    Vive 90 días sin actividad; preserva datos extraídos aún si los
    archivos originales se borraron a las 24h.
    """
    __tablename__ = "ict_sessions"

    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    user_id: Mapped[int] = mapped_column(
        ForeignKey("users.id", ondelete="CASCADE"), index=True, nullable=False
    )
    ejercicio_fiscal: Mapped[str] = mapped_column(String(4), nullable=False)
    ruc: Mapped[str] = mapped_column(String(20), nullable=False)
    razon_social: Mapped[str] = mapped_column(String(255), nullable=False)
    numero_adhesivo: Mapped[str | None] = mapped_column(String(64), nullable=True)

    status: Mapped[str] = mapped_column(
        String(20), default="in_progress", nullable=False
    )  # "in_progress" | "completed" | "expired"

    created_at: Mapped[datetime.datetime] = mapped_column(DateTime, nullable=False)
    last_activity_at: Mapped[datetime.datetime] = mapped_column(DateTime, nullable=False)
    expires_at: Mapped[datetime.datetime] = mapped_column(DateTime, nullable=False)
    # Renueva expires_at = last_activity_at + 90 días en cada update
```

### 4.2 ICTAnexo (NEW)

```python
class ICTAnexo(Base):
    """Datos extraídos por anexo. Persiste 90 días con la sesión."""
    __tablename__ = "ict_anexos"

    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    session_id: Mapped[int] = mapped_column(
        ForeignKey("ict_sessions.id", ondelete="CASCADE"), index=True, nullable=False
    )
    anexo_code: Mapped[str] = mapped_column(String(16), nullable=False)
    # Valores válidos en MVP: "INDICE", "A1", "A9"
    # Valores futuros: "A2", "A3", "A4", "A5", "A6", "A7", "A8"

    status: Mapped[str] = mapped_column(String(20), default="empty", nullable=False)
    # "empty" | "partial" | "ready" | "error"

    extracted_data: Mapped[dict | None] = mapped_column(JSON, nullable=True)
    # JSON dinámico por anexo:
    # A1: {"f101": {"311": 5000, "312": 1200, ...}, "balance": {"1.1.1.01.01": {nombre: "Caja", saldo: 4500}, ...}}
    # A9: {"f101": {"7001": 10000, "7010": 8000, ...}, "kardex": {nota: "no provisto"}}
    # INDICE: {"anexos_aplica": {"A1": "SI", "A9": "SI", "A2": "NO", ...}}

    warnings: Mapped[list[str] | None] = mapped_column(JSON, nullable=True)
    # ["Casillero 828 no detectado en el PDF F-101"]

    uploaded_files: Mapped[dict | None] = mapped_column(JSON, nullable=True)
    # Metadata de los archivos subidos (no el contenido):
    # {"f101": {filename: "F101_2025.pdf", size: 524288, uploaded_at: "2026-..."}}

    last_updated_at: Mapped[datetime.datetime] = mapped_column(DateTime, nullable=False)

    __table_args__ = (
        UniqueConstraint("session_id", "anexo_code", name="uq_session_anexo"),
    )
```

### 4.3 Sin cambios en otras tablas

`User`, `ClientDevice`, `ToolJob` quedan como están. El ICT es independiente del modelo `ToolJob` genérico (porque su flujo es distinto: persistente, no one-shot).

## 5. Endpoints HTTP

Todos prefijados `/api/v1/client/ict/`, protegidos por `require_client_with_device`.

| Método | Ruta | Qué hace |
|--------|------|----------|
| `POST` | `/sessions` | Crear nueva sesión ICT (`ruc`, `razon_social`, `ejercicio_fiscal`, `numero_adhesivo?`). Si el cliente ya tiene una `in_progress`, devuelve esa. |
| `GET` | `/sessions/active` | Devuelve la sesión activa del cliente o 404 si no hay. |
| `GET` | `/sessions/{id}` | Detalle: cliente_data + lista de anexos con su estado, warnings, archivos subidos. |
| `PATCH` | `/sessions/{id}` | Actualizar datos del contribuyente (razón social, adhesivo, etc.). |
| `POST` | `/sessions/{id}/anexos/{anexo_code}/upload` | Multipart: campo `slot_name` (ej. "f101", "balance", "kardex") + archivo. Procesa al instante. |
| `DELETE` | `/sessions/{id}/anexos/{anexo_code}/upload/{slot_name}` | Quita un archivo de un slot (reset el extracted_data parcial). |
| `GET` | `/sessions/{id}/download` | Genera Excel ICT al vuelo + StreamingResponse. |
| `DELETE` | `/sessions/{id}` | Marca la sesión como `expired` (cliente "termina" o "descarta"). |

## 6. Flujo de procesamiento por anexo

### 6.1 Mapping celda Excel → dato (cell_maps)

Cada anexo declara un dict que dice qué celda del template recibe qué dato.

**Ejemplo simplificado `cell_maps/a1.py`:**

```python
A1_SHEET = "MAPEO DE LA DECLARACIÓN A1"

# Encabezado del anexo (datos del cliente)
A1_HEADER_MAP = {
    "C5":  ("session", "razon_social"),
    "C6":  ("session", "ruc"),
    "C7":  ("session", "ejercicio_fiscal"),
}

# Casilleros F-101 que van en columna C (Valor declarado)
# Estructura: {fila_excel: numero_casillero}
A1_CASILLEROS_F101 = {
    15: "311",  # Efectivo y equivalentes
    16: "312",
    17: "313",
    # ... ~140 casilleros más
}

# Cuentas contables que van en columnas D-F (código, nombre, saldo)
# El catálogo dice qué prefijos contables pertenecen a cada casillero
# (ver mapping_catalog.py). Esto se calcula en tiempo de fill.
```

```python
from backend.app.ict.cell_maps.a1 import A1_SHEET, A1_HEADER_MAP, A1_CASILLEROS_F101
from backend.app.ict.mapping_catalog import F101_TO_BALANCE_PREFIXES


class A1Filler:
    anexo_code = "A1"

    def fill(self, workbook, session_data, anexo_data):
        ws = workbook[A1_SHEET]
        warnings = []
        filled = 0

        # 1. Llenar encabezado
        for cell, (source, key) in A1_HEADER_MAP.items():
            value = session_data.get(key, "")
            ws[cell] = value
            filled += 1

        # 2. Llenar casilleros F-101 (columna C)
        f101_data = anexo_data.get("f101", {})
        for row, casillero in A1_CASILLEROS_F101.items():
            value = f101_data.get(casillero)
            if value is not None:
                ws[f"C{row}"] = value
                filled += 1
            else:
                warnings.append(f"Casillero F-101 {casillero} no detectado")

        # 3. Llenar cuentas contables (columnas D-F)
        # Para cada casillero, buscar las cuentas del balance que matchen
        balance_data = anexo_data.get("balance", {})  # {código: {nombre, saldo}}
        for row, casillero in A1_CASILLEROS_F101.items():
            prefijos = F101_TO_BALANCE_PREFIXES.get(casillero, [])
            matched_cuentas = [
                (codigo, cuenta) for codigo, cuenta in balance_data.items()
                if any(codigo.startswith(p) for p in prefijos)
            ]
            if matched_cuentas:
                # El template tiene N filas en blanco para sub-cuentas
                # Llenamos secuencialmente desde la fila base
                for offset, (codigo, cuenta) in enumerate(matched_cuentas):
                    target_row = row + offset
                    ws[f"D{target_row}"] = codigo
                    ws[f"E{target_row}"] = cuenta["nombre"]
                    ws[f"F{target_row}"] = cuenta["saldo"]
                    filled += 3

        return {"filled_cells": filled, "warnings": warnings}
```

```python
ALL_F101_CASILLEROS = [
    # Bloque Estado Situación Financiera
    "311", "312", "313", "314", "315", "316", "317", "318", "319", "320",
    # ... muchos más
    # Bloque Inventarios A9
    "7001", "7010", "7013", "7022", "7025", "7028", "7031", "7034", "7037",
    # ... etc
]


def parse_f101(pdf_bytes: bytes) -> dict:
    """Devuelve {periodo, casilleros: {num: valor}, errores: []}."""
    try:
        with pdfplumber.open(BytesIO(pdf_bytes)) as pdf:
            text = "\n".join(p.extract_text() or "" for p in pdf.pages)
    except Exception as e:
        return {"periodo": None, "casilleros": {}, "errores": [f"PDF inválido: {e}"]}

    if not text.strip():
        return {"periodo": None, "casilleros": {}, "errores": ["PDF sin texto extraíble (¿escaneado?)"]}

    periodo = find_periodo(text)
    casilleros = {}
    errores = []
    for num in ALL_F101_CASILLEROS:
        v = find_casillero_value(text, num)
        if v is not None:
            casilleros[num] = v

    return {"periodo": periodo, "casilleros": casilleros, "errores": errores}
```

```python
from io import BytesIO
from openpyxl import load_workbook


def parse_balance(excel_bytes: bytes) -> dict:
    """Lee Balance de Comprobación. Devuelve {cuentas: {código: {nombre, saldo}}, errores}.

    Asume columnas: Código | Nombre | Saldo Final (otros formatos requieren mapeo flexible).
    Detecta automáticamente la fila de encabezado por keywords.
    """
    wb = load_workbook(BytesIO(excel_bytes), data_only=True)
    ws = wb.active

    # Detectar encabezado (busca fila con "código" y "saldo" / "monto")
    header_row = _find_header_row(ws)
    if header_row is None:
        return {"cuentas": {}, "errores": ["No se detectó fila de encabezado. Esperaba 'Código' y 'Saldo'."]}

    # Identificar índices de columnas
    col_codigo, col_nombre, col_saldo = _identify_columns(ws, header_row)
    if not all((col_codigo, col_saldo)):
        return {"cuentas": {}, "errores": ["No se identificaron columnas obligatorias."]}

    cuentas = {}
    errores = []
    for row in ws.iter_rows(min_row=header_row + 1, values_only=True):
        codigo = str(row[col_codigo]).strip() if row[col_codigo] else None
        if not codigo:
            continue
        nombre = str(row[col_nombre]).strip() if col_nombre is not None and row[col_nombre] else ""
        try:
            saldo = float(row[col_saldo]) if row[col_saldo] is not None else 0.0
        except (ValueError, TypeError):
            errores.append(f"Saldo inválido en fila código={codigo}")
            continue
        cuentas[codigo] = {"nombre": nombre, "saldo": saldo}

    return {"cuentas": cuentas, "errores": errores}
```

### 8.1 Errores de upload

| Caso | HTTP | Mensaje al cliente | Acción |
|------|------|--------------------|--------|

| Caso | Comportamiento |
|------|----------------|

### A.2 A1 — Patrón de inserción dinámica de filas

**Observación del ARCOLANDS real**: cada casillero F-101 en A1 tiene:
- 1 fila base con `casillero | nombre | valor_declarado | (cuenta base) | (saldo base) | (diff)`
- N filas hijas con subcuentas adicionales `| | | (cuenta) | (nombre) | (saldo) | |`

**Ejemplo casillero 311 EFECTIVO Y EQUIVALENTES** ($5,008,023.09 declarado):

```
r13: 311 | EFECTIVO Y EQUIVALENTES AL EFECTIVO | 5008023.09 | 1.1.01.01.01 | CAJA CHICA UIO | 300 | 0
r14:     |                                     |            | 1.1.01.01.02 | CAJA CHICA SSFD | 1000 |
r15:     |                                     |            | 1.1.01.01.04 | CAJA CHICA MDC-40 | 100 |
r16:     |                                     |            | 1.1.01.02.01 | BANCO PICHINCHA | 280330.14 |
r17:     |                                     |            | 1.1.01.02.02 | BANCO INTERNACIONAL | 3632210.06 |
r18:     |                                     |            | 1.1.01.02.03 | BANCO INTERNACIONAL CONTRATO | 1094082.89 |
```

**Algoritmo del A1Filler:**

```python
def fill(self, workbook, session_data, anexo_data):
    ws = workbook[A1_SHEET]
    f101_data = anexo_data.get("f101", {})          # {casillero: valor}
    balance_data = anexo_data.get("balance", {})    # {codigo: {nombre, saldo}}

    # Empezamos en la fila base del primer casillero (ej. fila 13)
    current_row = A1_FIRST_DATA_ROW  # ej. 13

    for casillero, casillero_nombre in F101_A1_CASILLEROS:
        valor_declarado = f101_data.get(casillero, 0)
        prefijos = F101_TO_BALANCE_PREFIXES.get(casillero, [])

        # Buscar subcuentas matching del balance
        matching_cuentas = sorted([
            (codigo, info) for codigo, info in balance_data.items()
            if any(codigo.startswith(p) for p in prefijos)
        ])

        # Escribir fila base
        ws[f"A{current_row}"] = casillero
        ws[f"B{current_row}"] = casillero_nombre
        ws[f"C{current_row}"] = valor_declarado

        # Si hay subcuentas, primera va en la fila base
        if matching_cuentas:
            codigo, info = matching_cuentas[0]
            ws[f"D{current_row}"] = codigo
            ws[f"E{current_row}"] = info["nombre"]
            ws[f"F{current_row}"] = info["saldo"]
            # Columna G ya tiene fórmula =F-C (no la tocamos)

            # Resto de subcuentas requieren insert_rows
            for offset, (codigo, info) in enumerate(matching_cuentas[1:], start=1):
                ws.insert_rows(current_row + offset)
                ws[f"D{current_row + offset}"] = codigo
                ws[f"E{current_row + offset}"] = info["nombre"]
                ws[f"F{current_row + offset}"] = info["saldo"]

            current_row += len(matching_cuentas)  # avanzar tras subcuentas
        else:
            current_row += 1
```
